# PoC Data Analysis
This project simulates a Proof of Concept (POC) for a large manufacturing company undergoing a procurement digitalization initiative.

The client’s Procurement department is facing challenges related to supplier master data quality, including:
- Duplicate supplier records
- Inconsistent naming conventions
- Outdated company information
- Limited spend visibility caused by fragmented supplier data

The dataset contained supplier input records together with multiple candidate company matches generated by the entity resolution engine.

# 1. Dataset Review
The first step of the project was understanding the structure of the provided dataset before validating any entity matches.

To better understand the data structure, I separated:
- Original supplier input attributes
- Candidate/enriched company attributes

I have used Excel for this part of the process and highlighted certain columns that I find to be useful for the matching process.

## Initial Observations
- Each supplier input row could contain multiple potential entity matches.
- Website domains and geographic consistency appeared to be strong validation signals.
- Some supplier records contained incomplete information, increasing ambiguity.
